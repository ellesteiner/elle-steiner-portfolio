EdTech: Student Polling App

Real-Time Classroom Engagement Redesign

Role: Lead Product Designer  //  Timeline: 9 months  //  Team: 1 PM, 5 engineers

Impact at a Glance

2× faster task completion (45s → 22s average)

100% success rate on core flows (up from 62%)

85% reduction in reported instructor anxiety

The Challenge

The original classroom polling tool created cognitive overload for instructors. Question creation required navigating an 8-step "module" builder, and a bloated creation modal displayed every possible question type simultaneously, most of which were rarely used. During live sessions, instructors reported high anxiety due to unclear system status and confusing controls. Usability testing revealed only 62% of core tasks were completed successfully, with an average completion time of 45 seconds for basic question creation.

My Approach

Research & Problem Discovery

I conducted a comprehensive audit of the existing application, analyzing usage data and interviewing 3 instructors to understand pain points. Three critical problems emerged:

Problem 1: Cognitive Overload in Question Creation. The modal presented 15+ question type options upfront, but data showed 80% of instructors only used 3 types (multiple choice, short answer, true/false). This created decision paralysis and slowed creation time.

Problem 2: Unclear System Status During Live Sessions. Instructors couldn't easily tell if questions were successfully delivered, if students were responding, or if the session was active. This ambiguity caused anxiety, especially during high-stakes classroom moments.

Problem 3: Excessive Steps to Launch. Getting from "I want to ask a question" to "students can now respond" required 8 clicks through multiple screens. For spontaneous classroom questions, this friction killed momentum.

Key Insight: Instructors needed two distinct modes: preparation mode (building question banks at their desk) and live mode (rapid-fire delivery in the classroom). The original design tried to serve both with one complex interface.

Design Strategy

Based on research findings, I established three design principles:

  • Default to the Common Case — Surface the 3 most-used question types prominently, hide advanced options
  • Clear System Status — Always show what's happening: preparing, live, responding, complete
  • Minimize Clicks to Launch — Reduce the path from question creation to student delivery

Key Design Decisions

Decision 1: Streamlined Question Creation Flow

Problem: 8-step module builder with every option visible

Solution: 3-step focused flow with progressive disclosure

  1. Choose question type (only showing top 3 types, "More" link for others)
  2. Write question and answers
  3. Review and launch

Rationale: Task analysis showed instructors rarely changed default settings. By hiding advanced options behind progressive disclosure, I reduced cognitive load while maintaining flexibility.

Impact: Question creation time dropped from 45s to 22s average (51% faster)

Decision 2: Live Session Status Indicator

Problem: No clear signal of session state or student activity

Solution: Persistent status banner showing:

  • Active session indicator (green pulse animation)
  • Live response count updating in real-time
  • Clear "End Session" action

Rationale: Instructors reported anxiety stemmed from uncertainty. A persistent, glanceable status indicator provided the confidence they needed to focus on teaching rather than troubleshooting.

Impact: 85% reduction in reported anxiety during live sessions (post-deployment survey)

Decision 3: Dropdown to Prominent Button

Problem: 70% of users failed to find question type selector in dropdown

Solution: Replaced hidden dropdown with prominent card-based selection on main screen

Rationale: Card-based selection improved discoverability and allowed for visual differentiation between question types (icons + descriptions).

Impact: Discoverability increased to 100% in subsequent testing

Validation & Iteration

I created interactive prototypes and conducted three rounds of usability testing with 18 instructors:

Round 1 - Initial Prototype:

  • 62% task success rate
  • Users struggled to find question type selector (dropdown)
  • Confusion about whether questions were "live" or "saved"

Changes Made:

  • Replaced dropdown with card-based selection
  • Added clear live/draft status labels
  • Simplified navigation from 3 tabs to 2 primary views

Round 2 - Revised Prototype:

  • 88% task success rate
  • Users still uncertain about live session status
  • Requests for more visible response tracking

Changes Made:

  • Added persistent status banner
  • Implemented real-time response counter
  • Designed clear session start/end flows

Round 3 - Final Validation:

  • 100% task success rate across all core flows
  • Average task completion time: 22s (down from 45s baseline)
  • 95% of users rated the experience "easy" or "very easy"

Design Solution

Instructor Experience

Dashboard Redesign. Simplified the main view into two clear sections:

  • Question Bank: Browse and prepare questions for future use
  • Live Session: Active polling with real-time response tracking

Question Creation Modal. Reduced from cluttered multi-step process to focused 3-step flow:

  1. Choose type (card-based selection of 3 common types + "More" option)
  2. Compose question (streamlined form with only essential fields)
  3. Preview and launch (clear review screen with one-click launch)

Live Session Interface. Designed a focused view optimized for in-the-moment use:

  • Large, clear status indicator (active/paused/ended)
  • Real-time response count with progress visualization
  • Simple one-click controls (next question, end session)
  • Results preview without leaving the flow

Student Experience

Simplified Response Flow. Reduced steps from 5 to 3:

  1. Join session (one-time code entry)
  2. Answer question (clear, touch-friendly interface)
  3. Confirm submission (immediate feedback)

Clear Visual Feedback

  • Submitted answers show confirmation state
  • Waiting state displays clear messaging
  • Session end shows thank you message

Outcomes & Impact

Measurable Results

Efficiency:

  • 51% faster task completion (45s → 22s average)
  • 8 steps reduced to 3 for question launch
  • 100% success rate on core flows (up from 62%)

User Satisfaction:

  • 85% reduction in reported instructor anxiety
  • 95% rated experience "easy" or "very easy"
  • Net Promoter Score increased from 42 to 78

Scale:

  • Deployed to 15,000+ instructors
  • Used across 200+ universities
  • 2M+ student responses in first 6 months

What Made This Successful

  • User-Centered Research: Deep understanding of instructor anxiety and classroom context drove design decisions rather than feature requests.
  • Iterative Validation: Three rounds of testing with 18 users ensured solutions actually worked before development investment.
  • Focus on Constraints: Designing for the most common case (3 question types, basic flow) while accommodating edge cases through progressive disclosure.
  • Measurable Impact: Clear before/after metrics demonstrated value to stakeholders and informed future product decisions.

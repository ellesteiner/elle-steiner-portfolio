Campaign Data Dashboard

Exploring AI-Assisted Complex Data Visualization

The Challenge

Can AI tools accelerate dashboard design for complex, multi-stakeholder products? I tested this by creating a campaign analytics concept using AI for research simulation and information architecture, then applying UX expertise to refine outputs into a credible design.

Project Type: Design experiment  |  Duration: 3 weeks  |  Focus: AI-assisted workflow

The Challenge

Design a campaign analytics dashboard displaying voter data, volunteer metrics, ballot tracking, and performance insights for diverse users (field organizers, data directors, volunteer coordinators, state staff).

Traditional approach: 8-12 weeks of stakeholder interviews, card sorting, iterative prototyping
My experiment: Compress to 2 weeks using AI, then validate approach against real tools

AI-Assisted Process

What I Used AI For

1. Persona Generation

Prompted Claude to create 4 realistic user personas based on campaign operations. Generated daily decisions, key needs, and pain points.

Result: Plausible starting framework in 90 minutes vs. 8 hours manually

2. Information Architecture

Simulated card sorting to identify logical data groupings. Generated 6 primary categories: Today's Priorities, Voter Universe, Team Performance, Ballot Tracking, Trends, Reports.

Result: Reasonable IA foundation aligned with existing tool patterns

3. Visualization Selection

Asked AI to recommend chart types for different data scenarios. Got appropriate suggestions: tables + maps for targeting, progress bars for ballot returns, heat maps for volunteer coverage.

Result: Good starting point requiring human refinement for accessibility

Data Type Visualization Rationale
Voter targeting priorities Sortable table + map Users need to export lists AND see geographic distribution
Ballot return rates Progress bars + trend lines Shows both status & momentum
Volunteer activity Heat map by time/location Reveals coverage gaps & peak times
Turnout modeling Gauge + confidence interval Shows prediction with uncertainty
Contact attempts Stacked bar by result Shows volume & quality simultaneously

Key Design Decisions

Decision 1: Progressive Disclosure Strategy

Problem: Complex dashboards overwhelm users with data OR hide essential information.

Solution: Three-tier hierarchy

  • Tier 1: Critical metrics (always visible)
  • Tier 2: Context (one click away)
  • Tier 3: Deep analysis (expandable)

Example: "47% ballot return" → "↑3% vs. yesterday, ↓2% vs. target" → Full trend chart with demographics

Decision 2: Role-Based Smart Defaults

Problem: Different users need different entry points.

Solution: Personalized landing views

  • Field Organizer → Today's contact priorities
  • Data Director → Trends & analysis
  • Volunteer Coordinator → Team performance
  • State Staff → Multi-campaign overview

Rationale: Reduces setup friction while preserving customization.

Decision 3: Algorithm Transparency

Problem: Users distrust "black box" priority scores.

Solution: Methodology tooltips

  • "Priority score: turnout likelihood (40%), persuadability (30%), contact history (30%)"
  • Link to full documentation
  • Manual weighting adjustment option

Rationale: Transparency builds trust for high-stakes decisions.

What AI Did Well VS What Needed Human Expertise

AI Strengths

  • Rapid persona generation - 4 detailed personas in minutes
  • Pain point brainstorming - 15+ relevant challenges quickly
  • Visualization recommendations - Appropriate chart types for data
  • Pattern recognition - Identified common dashboard structures

Human Expertise Required

  • Strategic prioritization - Which users/features to focus on first
  • Interaction design - Transitions, loading states, error handling
  • Accessibility - Color contrast, screen readers, keyboard nav
  • Visual polish - Typography, spacing, component consistency

Mobile-First Design Approach

60% of field staff check data on mobile while canvassing

Challenge: How do we display complex voter data, prioritization logic, and actionable insights on a 375px screen while users are actively canvassing?

Outcomes

Deliverables:

  • Dashboard concept with 20+ components
  • Information architecture for 6 data categories
  • Mobile-responsive layouts
  • Visualization library with 8 chart types

Time Efficiency:

  • Research: 90 minutes (vs. 8 hours traditional)
  • Information architecture: 90 minutes (vs. 4 hours)
  • Visualization selection: 90 minutes (vs. 3 hours)
  • Total: 70% time reduction in early phases

Limitations: This is an exploratory concept using AI-simulated research. It has NOT been validated with real campaign staff and would require extensive user testing before implementation.
